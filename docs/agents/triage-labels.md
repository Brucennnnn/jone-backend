# Triage Labels

Canonical triage role mapping for GitHub Issues:

| Canonical role | GitHub label |
| --- | --- |
| `needs-triage` | `needs-triage` |
| `needs-info` | `question` |
| `ready-for-agent` | `ready-for-agent` |
| `ready-for-human` | `ready-for-human` |
| `wontfix` | `wontfix` |

Category labels:

| Canonical role | GitHub label |
| --- | --- |
| `bug` | `bug` |
| `enhancement` | `enhancement` |

Current repo labels include `question`, `wontfix`, and `ready-for-agent`. If the workflow starts applying `needs-triage` or `ready-for-human`, create those labels first instead of substituting unrelated labels.
